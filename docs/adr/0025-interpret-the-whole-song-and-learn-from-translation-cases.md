# Interpret the whole song and learn from translation cases

Concert Live Workflow 不以孤立字詞或逐行直譯產生繁中；每首歌先建立有來源的 Song Interpretation Brief，釐清敘事主體、核心主軸、情緒弧線、反覆意象、跨行關係與歧義，再以它評估既有譯文及產生獨立 AI 草稿。外部譯者如何處理相同意象、候選譯法、採用／拒絕理由、Roy 修正與最終核准結果會寫入 Roy Translation Notebook，供後續歌曲研究與翻譯使用；每筆預設只是帶有歌曲語境、來源與 reuse terms 的案例，不能把單一詞彙脫離全曲主軸後機械套用，也不能因為記錄了別人的譯法就忽略重用限制。只有官方固定譯名或 Roy 明確指定「以後固定這樣翻」才能成為 Translation Rule；相似案例反覆採用時系統可以提出升級建議，但不得自行建立永久規則，套用時仍須驗證歌手、敘事視角、主軸與意象相容。Lyrics Packet 必須引用當次 brief、使用過的 notebook cases 與任何 Translation Rule，讓 Roy 核准的是一個能說明整體原意的翻譯版本，而不只是看似逐字對應的句子集合。
